# Security Policy

## About This Project

Weekend-to-Release is a static educational repository containing only Markdown documentation and Mermaid diagrams. It does not include executable code, dependencies, or services.

## Reporting a Vulnerability

If you discover a security concern related to this repository (for example, accidentally committed secrets, a malicious link, or a supply-chain risk in a referenced tool), please report it responsibly:

1. **Do not** open a public issue.
2. Email **[ParkWardRR](https://github.com/ParkWardRR)** directly via GitHub private messaging or by opening a [GitHub Security Advisory](https://docs.github.com/en/code-security/security-advisories/working-with-repository-security-advisories/creating-a-repository-security-advisory) on this repository.
3. Include a clear description of the concern and steps to reproduce (if applicable).

## Response Timeline

- **Acknowledgement:** within 72 hours.
- **Resolution or update:** within 14 days of acknowledgement.

## Scope

Because this repository contains no running software, the most likely security issues are:

- Leaked credentials or secrets in commit history.
- Malicious or compromised links in documentation.
- Social engineering risks in referenced third-party tools.

Thank you for helping keep this project safe.
